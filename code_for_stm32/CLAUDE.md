# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build

This is an STM32F103C8Tx (Cortex-M3, 64KB Flash, 20KB RAM) firmware project generated by STM32CubeMX. Requires `arm-none-eabi-` toolchain on PATH.

```bash
# Configure
cmake --preset Debug

# Build
cmake --build build/Debug

# Flash (ST-Link)
st-flash write build/Debug/code_for_stm32.elf 0x08000000
```

A secondary Clang-based toolchain config exists at `cmake/starm-clang.cmake` but is not the default.

## Architecture

The build links two targets into the final executable:

- **`stm32cubemx`** (INTERFACE library) — HAL include paths and compile definitions (`USE_HAL_DRIVER`, `STM32F103xB`).
- **`STM32_Drivers`** (OBJECT library) — STM32 HAL driver source files compiled once and linked into the executable.

Application sources (`Core/Src/*.c`) plus the startup assembly (`startup_stm32f103xb.s`) are added directly to the executable target by the CMakeLists in `cmake/stm32cubemx/`.

## Code generation (STM32CubeMX)

This project was generated by STM32CubeMX. Generated code is between `/* USER CODE BEGIN */` / `/* USER CODE END */` markers. Code outside those markers is regenerated when CubeMX re-generates the .ioc file. Add custom code only between the markers, or in the `Lib/` directory (which is entirely user-managed).

## Custom libraries

- **`Lib/PID/`** — PID controller library. Provides `pid_create`, `pid_step`, `pid_clear`, `pid_set_limit`, `pid_destroy`. Two PID instances (`pid_x`, `pid_y`) are created in `main.c` for 2-axis gimbal control but are not yet wired into the main loop.
- To add a new library, place headers in `Lib/<name>/Inc/` and sources in `Lib/<name>/Src/`, then add them to `CMakeLists.txt` in the `target_sources` and `target_include_directories` blocks.

## Peripherals

- **USART1, USART2, USART3** — all configured at 115200 baud, 8N1
- **PC13** — GPIO output (onboard LED on many STM32F103 boards)
- **HSE** — external high-speed oscillator, PLL at ×9 multiplier
